\section{Présentation de la structure}
La partie C contient :
\begin{itemize}
\item gencode : package gérant la structure générale du programme en assembleur
\item gestion\_registres : package permetant la gestion des ressources : allocation, libération...
\item outils\_instruction : package permettant de formatter le texte assembleur : instruction, commentaire...
\item outils\_parcours : package gérant le parcours dans l'arbre et la traduction des différentes instructions (passe 2 notamment)
\end{itemize} 


gencode et outils\_instruction utilisent les autres packages et donc dépendent de ces derniers.

\section{Définition des incréments}
La partie C ne contient qu'une passe pour le langage mini déca. Ainsi, nous avons décidé de faire une découpage suivant la grammaire : 
\begin{itemize}
\item print et println
\item littéraux
\item opération arithmétique et booléennes
\item opération d'entrée (lecture)
\item variable
\item structure de controle
\end{itemize}

\section{Méthode de tests}
A l'aide de la test suite, nous avons créer des programmes de tests permettant de tester chaque fonction par incréments. Nous avons vérifier l'utilisation et la libération des registres.
